# Inventory Manual QA Quick Check (10–15 мин)

Дата: 2026-03-08

Цель: быстро закрыть пункты `BLOCKED` из отчета QA.

## 0) Подготовка (1 мин)
1. Открыть приложение для `novikov_bh`.
2. Войти в inventory-режим.
3. Убедиться, что дата выставлена на сегодня.

## 1) B1/B2 — Добавление и Enter-сохранение (2–3 мин)
1. Нажать `+ Add row`.
2. Заполнить обязательные поля в строке.
3. Нажать `Enter` в одном из полей строки.
Ожидаемо:
- строка сохраняется,
- появляется в таблице без ошибок,
- номер строки (`#`) назначается корректно.

## 2) B5 — Edit/Delete/Save/Cancel (2–3 мин)
1. Нажать `Edit` у сохраненной строки.
2. Изменить одно поле и нажать `Save`.
3. Снова `Edit`, изменить поле и нажать `Cancel`.
4. Нажать `Delete` у строки.
Ожидаемо:
- `Save` сохраняет изменения,
- `Cancel` откатывает незасейвленные изменения,
- `Delete` удаляет строку из текущего дня.

## 3) E3 — Проверка console на i18n предупреждения (1–2 мин)
1. Открыть DevTools Console.
2. Перезагрузить страницу и открыть inventory.
Ожидаемо:
- нет неожиданных предупреждений вида `[Inventory i18n] Missing keys ...`,
- допустимы только заранее согласованные предупреждения (если есть).

## 4) F1 — Проверка XLSX экспорта (2 мин)
1. Добавить минимум 1 строку в диапазоне.
2. Нажать `Download XLSX`.
3. Открыть скачанный файл.
Ожидаемо:
- файл скачан без ошибки,
- лист существует,
- данные соответствуют строкам в inventory.

## 5) G2 — Back navigation (1 мин)
1. Находясь в inventory, нажать кнопку Back.
Ожидаемо:
- возврат в ожидаемый экран (без смешения UI режимов).

## 6) G3 — Поиск и inventory (1–2 мин)
1. Перейти между wine/menu и inventory.
2. Использовать поиск в wine-режиме.
3. Вернуться в inventory.
Ожидаемо:
- поиск не ломает inventory,
- inventory отображается корректно после переключений.

## 7) Как отметить результат в отчете
Файл отчета: `docs/inventory/INVENTORY_QA_RUN_2026-03-08.md`

Обновить статусы:
- `B1`, `B2`, `B5`, `E3`, `F1`, `G2`, `G3`: `PASS` или `FAIL`.
- Если есть `FAIL`: добавить запись в раздел дефектов с шагами воспроизведения.

## 8) Критерий GO
GO возможен, если:
- все перечисленные `BLOCKED` пункты переведены в `PASS`,
- нет новых дефектов уровня High/Critical.
