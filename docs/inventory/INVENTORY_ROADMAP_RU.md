## 2) Текущий статус

#### Цель
Убрать английские хардкод-ключи колонок в `exportXlsx()` и вынести их в RU/EN словари, чтобы экспорт соответствовал выбранному языку интерфейса.

#### Выполнено
1. **Расширить словарь `UI_STRINGS.inventory`**
   - Добавить ключи для заголовков экспорта:
     - `xlsxDate`, `xlsxRecord`, `xlsxProduct`, `xlsxQuantity`, `xlsxUnit`, `xlsxReason`, `xlsxStaff`, `xlsxConfirmedBy`.
   - Добавить RU и EN значения.

2. **Обновить `assets/js/inventory-mode-module.js`**
   - В `exportXlsx()` формировать `payload` через ключи `L.xlsx*`, а не через `Date/Record/...`.
   - Сохранить обратную совместимость через `defaultStrings` fallback.

3. **Проверка целостности экспорта**
   - Проверить, что:
     - структура файла не ломается,
     - лист создается корректно,
     - колонки в экспортируемом файле отображаются на выбранном языке.

4. **Точечная регрессия inventory**
   - Проверить, что изменения не затронули:
     - сохранение строк,
     - подтверждение дня,
     - рендер истории,
     - feature-gating.

#### Критерии готовности (DoD)
- В `exportXlsx()` нет хардкода заголовков колонок.
- При `state.lang = 'ru'` экспорт содержит русские заголовки.
- При `state.lang = 'en'` экспорт содержит английские заголовки.
- Диагностика редактора без новых ошибок.

## 6) QA-чеклист ручной приемки (Inventory Mode)

### A. Доступность фичи по флагу
- Для ресторана с `features.inventory.enabled = true` маршрут inventory открывается.
- Для ресторана с `features.inventory.enabled = false` inventory не отображается в доступных режимах.

### B. Базовый CRUD и валидация
- Добавление строки работает через кнопку `+ Add row` / `+ Добавить строку`.
- Сохранение строки по `Enter` в полях строки работает корректно.
- Валидация обязательных полей блокирует сохранение и показывает сообщение.
- Некорректное количество (не число/меньше 0) не сохраняется.
- Кнопки `Edit/Delete` и `Save/Cancel` отрабатывают без потери данных.

### C. Подтверждение дня
- Без фамилии подтверждающего подтверждение дня невозможно.
- Без строк за день подтверждение дня невозможно.
- После подтверждения день блокируется от редактирования/добавления.

### D. История и диапазон
- Фильтр `From/To` корректно ограничивает отображаемые записи.
- При пустом диапазоне отображается локализованное пустое состояние.

### E. Локализация UI
- При `state.lang = 'en'` все inventory-тексты в интерфейсе на английском.
- При `state.lang = 'ru'` все inventory-тексты в интерфейсе на русском.
- В консоли нет предупреждений i18n по missing/extra/placeholder mismatch (или они ожидаемы и разобраны).

### F. XLSX-экспорт
- Файл выгружается в диапазоне с данными.
- Имя листа соответствует текущей локали (`exportSheet`).
- Заголовки колонок в XLSX локализованы (`xlsx*` ключи).

### G. Регрессия shell-логики
- Переключение между menu/inventory режимами не смешивает UI.
- Кнопка Back в inventory возвращает в ожидаемый экран.
- Поисковая строка не ломает inventory-режим.
